## 6. Bilinen Sorunlar / Sınırlamalar

1. **NocoDB UI `unique` flag DB-level constraint oluşturmuyor.** Pratikte sorun yok çünkü `upsert_record` lookup-then-INSERT yapıyor (kod katmanında garanti). Defense-in-depth için ileride NocoDB altındaki SQLite/Postgres'e direct `CREATE UNIQUE INDEX` atılabilir (low priority).

2. **NOCODB_BASE_URL plain HTTP** (port 80). Production için HTTPS önerilir; bir nginx/Caddy reverse proxy + Let's Encrypt eklemek gelecekteki iş. Şu an ihtiyaç yok (Cloud Run → VM trafiği GCP iç ağında).

3. **Cloud Shell'de `read -p` token paste** — Beyza'nın SSH client'ında paste prompt'u çalışmadı, `export MA_TOK="..."` ile çözüldü. Yeni session'da bu trick'i unutmayın.

4. **mind-agent token READ+WRITE birlikte.** ADR-001 READ/WRITE ayrımı öneriyor; mind-id `/satis` deploy'ı yapıldığında ayrı bir READ token üretilecek.

5. **CLAUDE.md'de v1.18.0 yazıyordu**, ama gerçek deploy v1.14.0'dı. Versiyon takibi şimdiye kadar yarım kalmış. Bu PR ile bumped to **v1.19.0**.
