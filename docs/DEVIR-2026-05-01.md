## 6. Bilinen Sorunlar / Sınırlamalar

1. **NocoDB UI `unique` flag DB-level constraint oluşturmuyor.** Live test ile kanıtlandı: doğrudan API'den 2 POST = 2 kayıt. Pratikte sorun yok çünkü `upsert_record` lookup-then-INSERT/PATCH yapıyor (kod katmanında garanti). Live idempotency kanıtlandı: aynı external_id ile 2 /task çağrısı → NocoDB'de 1 kayıt. Defense-in-depth için ileride NocoDB altındaki SQLite/Postgres'e direct `CREATE UNIQUE INDEX` atılabilir.

2. **NocoDB v2 records API ARRAY body bekliyor** — bizim ilk implementasyonumuz tek obje gönderiyordu (KM4MZ branch'inden geldi), 422 ERR_INVALID_PK_VALUE alıyorduk. Fix v1.20.0'da: `nocodb_client.create_record/update_record` artık `[fields]` array gönderiyor. 13/13 test geçiyor + canlı kanıtlandı.

3. **NOCODB_BASE_URL plain HTTP** (port 80). Production için HTTPS önerilir; bir nginx/Caddy reverse proxy + Let's Encrypt eklemek gelecekteki iş. Şu an ihtiyaç yok (Cloud Run → VM trafiği GCP iç ağında).

4. **Cloud Shell'de `read -p` ve `read -s` token paste sorunlu** — Beyza'nın SSH client'ında paste prompt'u çalışmadı, `export VAR="..."` ile çözüldü. Yeni session'da bu trick'i unutmayın.

5. **mind-agent token READ+WRITE birlikte.** ADR-001 READ/WRITE ayrımı öneriyor; mind-id `/satis` deploy'ı yapıldığında ayrı bir READ token üretilecek.

6. **Burak'ın orijinal kurulumu Firebase + OpenAI iki yerden de bozuktu.** Bu session'da ortaya çıkardık ve düzelttik:
   - Firebase service account key `bd45a8f6...` GitHub'a expose edildiği için Google revoke etmiş (key sahibi: `seymaakrs/mind-agent` repo)
   - OpenAI key Burak'ın kişisel hesabındaymış, hesap kapatıldığında key de kapanmış
   - **`mind-agent` Cloud Run servisi aslında Burak ayrıldıktan sonra kısmen çalışmıyordu** — sadece Firestore'a yazmayan tool'lar (image, video read) çalışıyordu, /task'ı tetikleyen her gerçek iş bozuktu

7. **Repo'da hâlâ credential dosyaları var** (`mind-agent/serviceAccount.json` + `gcp-service-account.json`). Aktif anahtar Secret Manager'da, repo'daki dosyalar Google tarafından zaten disable edilmiş — yani **şu an direkt zarar yok**, ama kullanıcı güvenlik temizliği adımını yapmadan repo herkes için göründüğü gibi kalır.

8. **OpenAI `mind-agent-prod-2026-05-01` key chat'te göründü.** Şu an çalışıyor; kullanıcı smoke yeşil olduğu için sonradan revoke edecek, yenisini yapacak.
